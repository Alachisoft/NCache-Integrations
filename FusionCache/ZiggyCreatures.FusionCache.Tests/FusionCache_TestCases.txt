OVERVIEW
--------
This document lists all test cases across the FusionCache test suite and describes
their scope. Tests are organized by test file / test class. Async and Sync variants
share the same logical scope; only the async versions are listed where both exist.

--------------------------------------------------------------------------------
1. L1 (MEMORY CACHE) TESTS  [L1Tests_Async.cs / L1Tests_Sync.cs]
   Scope: Tests for single-layer, in-memory (L1) caching behavior only.
   No distributed cache or backplane involved.
--------------------------------------------------------------------------------

  TC-L1-01  CanRemoveAsync
            Verifies that a cached entry can be removed and is no longer returned.

  TC-L1-02  ReturnsStaleDataWhenFactoryFailsAsync
            Confirms stale data is returned when the factory throws and fail-safe is enabled.

  TC-L1-03  ReturnsStaleDataWhenFactoryFailsWithoutExceptionAsync
            Confirms stale data is returned when the factory returns a failure signal (no exception).

  TC-L1-04  ThrowsWhenFactoryThrowsWithoutFailSafeAsync
            Ensures an exception propagates to the caller when fail-safe is disabled and factory fails.

  TC-L1-05  ThrowsOnFactoryHardTimeoutWithoutStaleDataAsync
            Verifies that a hard timeout exception is thrown when no stale data is available.

  TC-L1-06  ReturnsStaleDataWhenFactorySoftTimeoutWithFailSafeAsync
            Validates that soft-timeout triggers background refresh and returns stale data immediately.

  TC-L1-07  DoesNotSoftTimeoutWithoutStaleDataAsync
            Ensures soft-timeout does not activate if there is no stale data to fall back to.

  TC-L1-08  DoesHardTimeoutEvenWithoutStaleDataAsync
            Verifies hard timeout fires even when no stale value exists in cache.

  TC-L1-09  ReturnsStaleDataWhenFactoryHitHardTimeoutWithFailSafeAsync
            Checks stale data is returned on hard timeout when fail-safe is active.

  TC-L1-10  HandlesBackgroundFactoryExceptionAsync
            Ensures background factory exceptions are swallowed or handled gracefully.

  TC-L1-11  SetOverwritesAnExistingValueAsync
            Confirms that calling Set replaces a previously cached value.

  TC-L1-12  GetOrSetDoesNotOverwriteANonExpiredValueAsync
            Validates GetOrSet returns the existing value without re-running the factory.

  TC-L1-13  DoesNotReturnStaleDataIfFactorySucceedsAsync
            Verifies fresh data from a successful factory replaces and is returned over stale data.

  TC-L1-14  GetOrDefaultDoesReturnStaleDataWithFailSafeAsync
            Checks GetOrDefault falls back to stale data when fail-safe is enabled.

  TC-L1-15  GetOrDefaultDoesNotReturnStaleDataWithoutFailSafeAsync
            Ensures GetOrDefault does not return stale data when fail-safe is disabled.

  TC-L1-16  FactoryTimedOutButSuccessfulDoesUpdateCachedValueAsync
            Confirms that a factory completing after a timeout still updates the cache.

  TC-L1-17  TryGetReturnsCorrectlyAsync
            Validates TryGet returns a hit/miss status and correct value.

  TC-L1-18  CanCancelAnOperationAsync
            Tests that cache operations can be cancelled via CancellationToken.

  TC-L1-19  HandlesFlexibleSimpleTypeConversionsAsync
            Checks that simple value types are correctly stored and retrieved.

  TC-L1-20  HandlesFlexibleComplexTypeConversionsAsync
            Verifies complex object types survive round-trip through the cache.

  TC-L1-21  GetOrDefaultDoesNotSetAsync
            Ensures GetOrDefault does not write any value to the cache.

  TC-L1-22  GetOrSetWithDefaultValueWorksAsync
            Validates GetOrSet with a default value returns that default on miss.

  TC-L1-23  ThrottleDurationWorksCorrectlyAsync
            Tests that the throttle duration prevents re-executing the factory too frequently.

  TC-L1-24  AdaptiveCachingAsync
            Validates that the factory can dynamically adjust cache entry options at runtime.

  TC-L1-25  AdaptiveCachingWithBackgroundFactoryCompletionAsync
            Checks adaptive caching behavior when the factory completes in the background.

  TC-L1-26  AdaptiveCachingDoesNotChangeOptionsAsync
            Confirms adaptive caching does not mutate the original default entry options.

  TC-L1-27  AdaptiveCachingCanWorkWithSkipMemoryCacheAsync
            Validates adaptive caching when memory cache writes are skipped.

  TC-L1-28  AdaptiveCachingCanWorkOnExceptionAsync
            Ensures adaptive caching handles factory exceptions without crashing.

  TC-L1-29  FailSafeMaxDurationNormalizationOccursAsync
            Verifies that fail-safe max duration is normalized relative to entry expiry.

  TC-L1-30  ReturnsStaleDataWithoutSavingItWhenNoFactoryAsync
            Checks that stale data is returned in read-only mode without re-saving it.

  TC-L1-31  CanHandleInfiniteOrSimilarDurationsAsync
            Tests cache behavior when duration is set to TimeSpan.MaxValue or similar.

  TC-L1-32  CanHandleZeroDurationsAsync
            Ensures zero-duration entries are handled without errors (immediate expiry).

  TC-L1-33  CanHandleNegativeDurationsAsync
            Verifies negative durations are treated as expired/invalid without errors.

  TC-L1-34  CanConditionalRefreshAsync
            Tests conditional refresh: factory only runs when ETag/modified condition changes.

  TC-L1-35  CanEagerRefreshAsync
            Validates eager refresh: cache refreshes in the background before full expiry.

  TC-L1-36  CanEagerRefreshWithInfiniteDurationAsync
            Confirms eager refresh works correctly when the duration is effectively infinite.

  TC-L1-37  CanEagerRefreshNoCancellationAsync
            Ensures eager refresh continues to completion and is not cancelled prematurely.

  TC-L1-38  NormalFactoryExecutionWaitsForInFlightEagerRefreshAsync
            Tests that a normal factory call blocks until an in-flight eager refresh completes.

  TC-L1-39  CanExpireAsync
            Verifies manual expiry marks an entry as stale/expired.

  TC-L1-40  CanSkipMemoryCacheAsync
            Checks that the SkipMemoryCache flag causes reads/writes to bypass L1.

  TC-L1-41  CanUseNullFusionCacheAsync
            Ensures the Null implementation of IFusionCache behaves safely (no-ops).

  TC-L1-42  FailSafeMaxDurationIsRespectedAsync
            Validates that stale data is not served beyond the fail-safe maximum duration.

  TC-L1-43  CanAutoCloneAsync
            Tests that AutoClone serializes/deserializes values to prevent mutation of cached objects.

  TC-L1-44  AutoCloneSkipsImmutableObjectsAsync
            Confirms AutoClone does not clone types marked as immutable.

  TC-L1-45  CanRemoveByTagAsync
            Verifies all entries associated with a given tag are removed together.

  TC-L1-46  CanRemoveByTagMultiAsync
            Validates bulk removal of entries associated with multiple tags.

  TC-L1-47  CanRemoveByTagWithChangingTagsAsync
            Checks that tag-based removal works even when tags on an entry have changed.

  TC-L1-48  CanRemoveByTagWithBehaviorExpireAsync
            Tests tag-based removal using the Expire behavior (marks entries as stale).

  TC-L1-49  CanRemoveByTagWithBehaviorRemoveAsync
            Tests tag-based removal using the Remove behavior (deletes entries entirely).

  TC-L1-50  CanClearAsync
            Validates that Clear removes all entries from the memory cache.

  TC-L1-51  CanClearWithFailSafeAsync
            Confirms Clear works correctly when fail-safe is active on entries.

  TC-L1-52  CanSkipMemoryCacheReadWriteAsync
            Tests combined skip of both read and write to memory cache.

  TC-L1-53  CanSoftFailWithSoftTimeoutAsync
            Verifies soft-fail behavior activates on soft timeout.

  TC-L1-54  CanDisableTaggingAsync
            Ensures tagging can be disabled without breaking normal cache operations.

  TC-L1-55  CanHandleEagerRefreshWithTagsAsync
            Tests that eager refresh properly preserves and propagates entry tags.

2. L1+L2 (MEMORY + DISTRIBUTED CACHE) TESTS  [L1L2Tests_Async.cs / L1L2Tests_Sync.cs]
   Scope: Two-layer cache (memory + distributed, e.g. Redis/NCache). No backplane.
--------------------------------------------------------------------------------

  TC-L2-01  ReturnsDataFromDistributedCacheIfNoDataInMemoryCacheAsync
            Validates that a cache miss in L1 falls through to L2 (distributed cache).

  TC-L2-02  HandlesDistributedCacheFailuresAsync
            Checks that distributed cache errors are handled gracefully (fail-safe / fallback).

  TC-L2-03  AppliesDistributedCacheHardTimeoutAsync
            Ensures hard timeout is enforced on distributed cache operations.

  TC-L2-04  AppliesDistributedCacheSoftTimeoutAsync
            Validates soft timeout behavior for distributed cache reads.

  TC-L2-05  DistributedCacheCircuitBreakerActuallyWorksAsync
            Tests that repeated distributed cache failures open the circuit breaker.

  TC-L2-06  DistributedCacheWireVersionPrefixModeWorksAsync
            Confirms wire version prefix is correctly added to distributed cache keys.

  TC-L2-07  DistributedCacheWireVersionSuffixModeWorksAsync
            Confirms wire version suffix is correctly added to distributed cache keys.

  TC-L2-08  DistributedCacheWireVersionNoneModeWorksAsync
            Validates that no wire version modifier is applied when mode is None.

  TC-L2-09  ReThrowsOriginalExceptionsAsync
            Ensures original exceptions from the factory are re-thrown to the caller.

  TC-L2-10  ReThrowsDistributedCacheExceptionsAsync
            Validates distributed cache exceptions propagate when configured to do so.

  TC-L2-11  ReThrowsSerializationExceptionsAsync
            Ensures serialization errors from L2 are surfaced as exceptions.

  TC-L2-12  SpecificDistributedCacheDurationWorksAsync
            Tests that a custom distributed cache TTL differs from the L1 TTL.

  TC-L2-13  SpecificDistributedCacheDurationWithFailSafeWorksAsync
            Confirms custom distributed TTL integrates correctly with fail-safe.

  TC-L2-14  DistributedCacheFailSafeMaxDurationWorksAsync
            Validates the fail-safe max duration cap applies to distributed entries.

  TC-L2-15  DistributedCacheFailSafeMaxDurationNormalizationOccursAsync
            Checks normalization of fail-safe max duration against distributed TTL.

  TC-L2-16  MemoryExpirationAlignedWithDistributedAsync
            Ensures L1 expiry is aligned with the distributed cache entry's actual expiry.

  TC-L2-17  CanSkipDistributedCacheAsync
            Tests that the SkipDistributedCache flag prevents all L2 interactions.

  TC-L2-18  CanSkipDistributedCacheOnReadOnlyAsync
            Validates skipping distributed cache reads during read-only operations.

  TC-L2-19  CanSkipDistributedReadWhenStaleAsync
            Confirms L2 read is skipped when stale data is acceptable (SkipDistributedReadWhenStale).

  TC-L2-20  DoesNotSkipOnMemoryCacheMissWhenSkipDistributedCacheReadWhenStaleIsTrueAsync
            Ensures L2 is still read on a full cache miss even when skip-stale-read is set.

  TC-L2-21  CanHandleConditionalRefreshAsync
            Tests conditional refresh logic with a simulated HTTP ETag/304 pattern.

  TC-L2-22  CanHandleEagerRefreshAsync
            Validates eager background refresh with a two-layer cache setup.

  TC-L2-23  EagerRefreshDoesNotBlockAsync
            Confirms eager refresh in L1+L2 mode does not block the calling thread.

  TC-L2-24  CanSkipMemoryCacheAsync
            Verifies L1 can be skipped so all reads/writes go directly to L2.

  TC-L2-25  CanExecuteBackgroundDistributedCacheOperationsAsync
            Tests that write-behind to L2 executes asynchronously in the background.

  TC-L2-26  FailSafeMaxDurationIsRespectedAsync
            Validates fail-safe max duration is enforced in the L1+L2 scenario.

  TC-L2-27  CanRemoveByTagAsync
            Checks tag-based removal propagates to the distributed cache layer.

  TC-L2-28  CanClearAsync
            Verifies cache clear removes entries from both L1 and L2.

  TC-L2-29  CanUseMultiNodeCachesWithSizeLimitAsync
            Tests multi-node cache operation when a size limit is applied to L1.
